## Как опубликовать

1. Создайте новый репозиторий на GitHub.
2. Загрузите **содержимое этой папки** в корень репозитория. Файл `index.html` должен лежать в корне, а не внутри дополнительной папки.
3. Откройте `Settings` → `Pages`.
4. В разделе `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку `main`, папку `/(root)` и нажмите `Save`.
6. Ссылка появится на этой же странице. Обычно она имеет вид `https://ИМЯ.github.io/РЕПОЗИТОРИЙ/`.

## Важное ограничение

GitHub Pages публикует только статические файлы и не запускает PHP. Поэтому эта версия хранит пользователей, уроки, задания и сообщения в `localStorage` текущего браузера.

Следствия:

- данные не синхронизируются между разными телефонами и компьютерами;
- очистка данных сайта в браузере удалит локальную базу;
- парольная защита здесь предназначена только для демонстрации, а не для настоящих секретных данных;
- исходные PHP-файлы, SQLite-база и хэши паролей специально не включены в эту папку.

Для общего кабинета с единой базой пользователей понадобится внешний сервер с PHP либо отдельный облачный backend.

## Резервная копия локальных данных

Откройте инструменты разработчика браузера и выполните:

```js
copy(AKASHA_LOCAL_TOOLS.exportData())
```

Для полного сброса локальной версии:

```js
AKASHA_LOCAL_TOOLS.resetData()
```
